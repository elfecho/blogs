Github pages + hexo 优雅地打造免费个人静态网页博客

前言：近期觉得写博客必须养成习惯，一方面养成做笔记做总结的习惯，同时也能做些生活记录，提高写作能力，其实有很多不错的博客平台都可以用，自然曝光率可能也比较高，但是咱好歹也是程序员，博客网站不自己搭建一个感觉也不太好，后来发现很多大牛的个人博客都在用hexo，折腾了一下觉得简单又实用，markdown编写git发布，静态网页速度快，稳定性高，还利于搜索引擎收录，对于博客程序来说这一切足够了。

发布你的博客

每次编写博客的步骤：
本地写好博客

1. 打开终端，cd到博客目录，执行 hexo clean 清空缓存
2. 执行 hexo g 构建文章页面
3. 执行 hexo s 本地预览整个站点
4. 执行 hexo d 发布到Pages更新线上博客

博客绑定自己的域名

域名解析CNAME记录到你仓库的默认域名，创建一个文本，写上你解析的域名，放在仓库根目录即可。

重新生成网页文件会把 CNAME 文件覆盖掉，需要直接把 CNAME 文件添加到 source 文件夹里，这样每次推的时候就不用担心仓库里的 CNAME 文件被覆盖掉了。
仓库设置中出现Domain’s DNS record could not be retrieved.提示说明CNAME文件出错，检查该文件域名是否拼写错误，注意不要加http前缀。
